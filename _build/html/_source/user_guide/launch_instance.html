Create Network, Subnet and router¶

  1. Login to Openstack Dashboard as normal user (demo).

  2. Navigate to Network. Select Network Topology or Networks.

  3. Click on “Create Network”. Enter the Network name and set Admin State UP. Click next

  4. Set the options Subnet Name, Network Address(CIDR Format), IP Version and Gateway IP. Click next

  5. Set additional details as per your requirement or leave blank. Click create.

  6. Once you have created Network, Network topology will look like below.

  7. Click on “Create router”. Enter Router Name, set Admin State UP and External Network VJTINet.

  8. Navigate to Routers. Click on the router you just created.

  9. Select the “Interfaces” tab and click on “Add Interface”.

  10. Select “Subnet”, IP Address for router interface(Optional). Click on Add Interface.

  11. Final Network Topology will look like below.

Create security group & configure the rules¶

  1. Navigate to Access & Security. Select the tab called “Security Groups”.

  2. Click on “Create Security group”. Enter the name and description for the security group.

  3. Once the group has been created successfully, Click on “Manage Rules”.

  4. Click on “Add Rule”.

  5. Allow ssh from anywhere to the instances (SSH).

  6. Similarly , allow “ping” as well to this host from anywhere (ALL ICMP).

    Once you have added those rules to the security group, it will look like below.

Create key-pair¶

To access the instance without password.

  1. Navigate to security & access. Click the tab called “Key Pairs” and click on “Create key Pair”.

  2. Enter the Key pair name. (Keep Some meaning full name). Click on “Create key Pair”.

  3. The key pair will be automatically downloaded to your laptop. If it didn’t download, click the link to download it. Keep the key safe since you can’t download it again.

Launch an instance¶

  1. Click on “Launch Instance ” button.

  2. Set the instance details like below.

    Availability Zone – nova . (Need to select your compute node).

    Instance Name – Enter the desired instance name

    Flavour – Select the available flavour according to your need. (See the details in right side)

    Instance Count – Enter the instance Count

    Boot Source – Select boot from pre-defined image.

    Image Name – select “cirros” since its very small Linux foot print for testing openstack.

  3. Click on Access & security tab for the instance. From the drop down box, select the key pair “demo-key” which we have created earlier. Also select the security group which we have created.

  4. Click on Networking tab, Select network by clicking on + sign.

  5. You can customise your instance after it has launched using the Post Creation (optional) option. Following is a example script to set password to user in Ubuntu Server and centOS cloud images.

    #cloud-config
password: mypassword
chpasswd: { expire: False }
ssh_pwauth: True

  6. Specify Advanced options (optional) to use when launching an instance. Click “Launch” to launch the new instance.

  7. Here you can see that instance has been launched. It will take few seconds/minutes to boot the instance depends on the image size which we have selected.

  8. If you would like to see the instance console, click the instance name and select the console tab. You should be able to access the instance here as well by double clicking the console bar.

  9. You can also check the log to know the instance is booted or not. (If console is not working)

  10. Assign floating ip to an instance so we can access it from external network.

  11. Add Floating IP from pool of IP addresses.

  12. Allocate IP.

  13. Associate to your VM.

  14. You can see floating IP in instance table.

  15. Now you can SSH into your VM using following.

ssh -i key.pem username@floating-ip-of-VM

for example

ssh -i demo-key.pem cirros@172.18.38.140
